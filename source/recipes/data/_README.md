# Recipes 数据维护说明

这个目录中的每个普通 `.md` 文件代表一道菜。构建网站时，`scripts/recipes-page.js` 会读取这些文件并生成 `/recipes/data.json`，页面本身不再维护菜品数据。

## 新增菜品

1. 复制 `_template.md`。
2. 将副本改成唯一的英文文件名，例如 `hong-shao-qie-zi.md`。
3. 填写顶部属性、食材、制作步骤、注意事项和下厨记录。
4. 执行 `npm run build`；本地编辑时也可以使用 `npm run server` 预览。

## 修改菜品

直接编辑对应的 Markdown 文件并重新构建。`slug` 是详情页 URL 使用的唯一标识，已有菜品不建议随意修改。

## 删除菜品

删除对应的 Markdown 文件并重新构建即可。以下划线开头的文件（如 `_template.md`、本文件）不会被当作菜品读取。

## 格式约定

- 食材：`- 名称 | 用量 | 预处理方式`
- 步骤：`1. **步骤标题**：具体做法`
- 步骤提示：紧跟步骤下一行填写 `- 提示：内容`
- 下厨记录：`- 日期 | 1 到 5 的评分 | 记录内容`
- `favorite: true` 会将菜品归入“常做”筛选。
- `updated` 使用 `YYYY-MM-DD` 格式，决定默认排序与页面的最近更新时间。
